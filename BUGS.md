# BUGS.md

## Баг-репорты

### BUG-1 (TC-POST-01): Несоответствие тела ответа при успешном POST запросе

**Краткое описание:** При успешном создании объявления тело ответа не соответствует ожидаемой JSON-схеме. Вместо структурированного объекта с полями объявления возвращается строка с сообщением.

**Шаги воспроизведения:**
1. Установить заголовок Content-Type: application/json
2. Установить заголовок Accept: application/json
3. Отправить POST запрос на https://qa-internship.avito.com/api/1/item с телом, заполненным валидными данными

**Фактический результат:**
{"status": "Сохранили объявление - ec71566b-6042-4b76-bb7a-f85add52f290"}

**Ожидаемый результат:**
Статус ответа 200 OK. Тело ответа соответствует JSON-схеме:
{"id": "<string>", "sellerId": "<integer>", "name": "<string>", "price": "<integer>", "statistics": {"likes": "<integer>", "viewCount": "<integer>", "contacts": "<integer>"}, "createdAt": "<string>"}

**Серьёзность:** Major

**Окружение:** Postman Version 11.91.0

## Замечания

1. **Некорректный статус-код при успешном POST запросе.** При успешном создании объявления должен приходить статус-код 201 (Created), а не 200 (OK).

2. **Неинформативное сообщение об ошибке при некорректном вводе.** При неверном формате данных тело ответа содержит неоднозначное сообщение "не передано тело объявления", что не позволяет понять, какая именно переменная введена некорректно.

3. **Некорректное сообщение об ошибке при sellerID = 0 или пустой строке.** При передаче sellerID = 0 (или пустой строки для строковых полей) возвращается сообщение "поле sellerID обязательно", хотя поле фактически присутствует в запросе.

4. **Отсутствие обработки статус-кода 404 для GET запросов объявлений продавца.** В документации и реализации отсутствует описание и обработка ситуации, когда объявления продавца не найдены.

5. **Путаница в документации с именованием поля продавца.** В документации одновременно используются sellerId и sellerID, что вызывает путаницу при написании тестов и интеграции.

6. **Неполнота документации API.** Документация не содержит четких спецификаций по ряду сценариев, что привело к необходимости помечать некоторые тест-кейсы как "уточнить у аналитика". В частности, не определено ожидаемое поведение при:
   - Отсутствии поля statistics в запросе (TC-POST-18)
   - Наличии лишних полей в запросе (TC-POST-20)
   - Передаче sellerID = 0 (TC-GET-SELLER-05)
   - Запросе объявлений с несуществующим sellerID (TC-GET-SELLER-06)
   
   Из-за отсутствия четких требований по этим сценариям соответствующие баги не были заведены, так как невозможно однозначно определить, какое поведение является корректным.
